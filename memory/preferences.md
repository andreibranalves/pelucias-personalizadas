# Preferences

## Communication
- Do not deploy to Vercel unless explicitly asked
- Keep responses short and direct — no summaries after edits
- Don't explain what was done unless something is non-obvious

## Code style
- Edit only what is asked — no extra refactoring or cleanup
- Prefer CSS overrides in media queries rather than restructuring base styles
- Single `index.html` file — no build tools, no separate CSS files

## Design preferences
- Dislikes childish/playful fonts (rejected Fredoka, Nunito)
- Prefers clean, professional look while keeping brand warmth
- Hard drop shadows (neobrutalism style) are part of the design language — keep them
- Does not want competitor names cited in comparison tables
- Prefers minimal CTAs — removed duplicate/redundant buttons

## Workflow
- User tests on real iPhone before approving changes
- User will ask for Vercel deploy explicitly when ready
- Reverts are taken seriously — don't reintroduce removed elements
